# 计算器历史记录功能 - PRD

## 1. 需求背景
用户希望在进行多次计算时，能够查看最近的历史记录，以便核对或复用之前的结果。

## 2. 功能目标
- **后端**: 增加 `GET /api/history` 接口，返回最近 5 条计算记录。每次计算成功后，将记录存入内存队列。
- **前端**: 在计算器界面下方增加一个“历史记录”区域，展示最近 5 条记录（格式如 `1 + 1 = 2`）。

## 3. 验收标准
1.  **存储逻辑**: 计算成功后立即保存，最多保留 5 条，超过 5 条时移除最早的一条。
2.  **展示逻辑**: 每次计算后或页面加载时，自动刷新历史记录列表。
3.  **异常处理**: 如果计算出错（如除以0），不应保存到历史记录中。
