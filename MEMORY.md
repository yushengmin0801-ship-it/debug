# MEMORY.md

## 标准开发工作流 (Standard Iteration Workflow - 12 Nodes)
所有软件开发项目必须严格遵守以下 12 个节点，并在每个节点完成时向 Stakeholder 发送进度消息：

1.  **需求确认**: 产品经理完成需求文档 -> @UI设计 (交付UI)
2.  **UI交付**: UI设计完成设计图 -> @后端开发 (交付后端)
3.  **后端交付**: 后端完成数据库设计、接口定义 -> @前端开发
4.  **联调开发**: 前端开发 + 接口联调
5.  **提测**: 前端开发完成 -> @测试 (提交测试)
6.  **一轮测试**: 测试进行第一轮测试，列出问题清单 -> @前后端开发
7.  **一轮修复**: 前后端修复问题 -> @测试 (通知回归)
8.  **二轮测试**: 测试进行第二轮测试及修复，直到无问题
9.  **全量回归**: 测试完成全量回归 -> @产品经理 (交付验收)
10. **产品验收**: 产品经理验收
11. **验收完成**: 产品验收通过 -> @项目经理 (通知PM)
12. **最终交付**: 项目经理反馈 **公网地址**
